# Getting Started

## Installation

```bash
npm install @heirshq/design-system
```

---

## Styling setup

The package ships two CSS exports that work together.

| Import | Purpose |
|---|---|
| `@heirshq/design-system/styles` | Required. Registers the `@source` path so Tailwind v4 includes component utility classes, and defines animation keyframes (Accordion, etc.). |
| `@heirshq/design-system/theme` | Optional but recommended. Provides default `@theme` token mappings and `:root` / `.dark` CSS variables so you only need to write the values you want to change. |

### Minimal setup (Tailwind v4)

Add these imports at the top of your global CSS file, before `@import "tailwindcss"`:

```css
@import "@heirshq/design-system/styles";
@import "@heirshq/design-system/theme";
@import "tailwindcss";
```

That's it. Every component will render correctly with the default Heirs brand tokens.

## Customising tokens

Because `@heirshq/design-system/theme` is imported first, any `@theme` or `:root`
block you declare afterwards **overrides it via the CSS cascade** — no `!important`,
no duplication required.

### Override the primary color scale

```css
@import "@heirshq/design-system/styles";
@import "@heirshq/design-system/theme";
@import "tailwindcss";

@theme {
  --color-primary-50:  #fef2f2;
  --color-primary-100: #fee2e2;
  --color-primary-200: #fecaca;
  --color-primary-300: #fca5a5;
  --color-primary-400: #f87171;
  --color-primary-500: #ef4444;
  --color-primary-600: #dc2626;
  --color-primary-700: #b91c1c;
  --color-primary-800: #991b1b;
  --color-primary-900: #7f1d1d;
  --color-primary-950: #450a0a;
}
```

### Override semantic tokens

Semantic tokens (background, foreground, radius, etc.) are raw CSS variables
defined in `:root`. Override them the same way:

```css
:root {
  --radius: 0.375rem;
  --primary: oklch(0.55 0.22 29);
  --background: oklch(0.99 0 0);
}

.dark {
  --primary: oklch(0.65 0.2 29);
  --background: oklch(0.12 0 0);
}
```

## Tailwind content scanning

The `@source` directive is embedded in `@heirshq/design-system/styles` and
points to the package's own `dist/` folder. **You do not need to add a manual
`@source` line for this package in your CSS.**

If you use Tailwind v3, add the content path in `tailwind.config.js` instead:

```js
// tailwind.config.js
module.exports = {
  content: [
    "./src/**/*.{ts,tsx}",
    "./node_modules/@heirshq/design-system/dist/**/*.js",
  ],
};
```
